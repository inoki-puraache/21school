# Портфолио: Frontend Developer

Этот репозиторий я использую как подборку практических работ по JavaScript, SQL и проектированию логики приложений.

Основной вектор моего развития — frontend-разработка. При этом мне важно не только собирать интерфейсы, но и хорошо понимать внутреннюю логику приложения, структуру данных и взаимодействие с API.

## Проекты

### [Механика игры «Морской бой»](./js%20tasks/README.md)
Небольшой JavaScript-проект с поэтапной реализацией игровой логики.

- ООП-модели `Ship`, `Board`, `Player`, `HumanPlayer`, `AIPlayer`, `App`
- обработка ходов, размещение кораблей, проверка победы
- развитие решения через несколько итераций

### [Заметки по проектированию Tic-Tac-Toe API](./js%20project/README.md)
Архитектурная заготовка backend-сервиса на `Node.js` и `NestJS`.

- разделение на `web`, `domain`, `datasource`
- моделирование игрового состояния
- проектирование API и модульной структуры

### [Базовые SQL-запросы](./sql%20tasks/README.md)
Подборка компактных запросов с фильтрацией, сортировкой и подзапросами.

- чистые `SELECT`-запросы
- диапазоны дат и многополевая сортировка
- вычисляемые поля и вложенные запросы

### [SQL: множества и JOIN-операции](./sql%20tasks%20part%202/README.md)
Примеры запросов с множествами, объединениями и реляционной логикой.

- `UNION`, `INTERSECT`, `EXCEPT`
- `CROSS JOIN`, `NATURAL JOIN`
- сравнение разных стратегий решения

### [Продвинутые JOIN-запросы и CTE](./sql%20+%20database/README.md)
SQL-подборка, ближе всего к прикладным backend- и analytics-сценариям.

- сборка понятных результирующих выборок из нескольких таблиц
