# Заметки по проектированию Tic-Tac-Toe API

В этой папке сейчас находится проектная заготовка и архитектурные заметки для небольшого backend-сервиса на `Node.js` и `NestJS`.

## Идея

Сервис должен принимать текущее состояние игры в крестики-нолики, валидировать ход пользователя и возвращать обновленное состояние после ответного хода сервера.

## На что сделан упор

- разделение на `web`, `domain` и `datasource` слои
- моделирование состояния игры
- валидация доски и правил хода
- абстракция репозитория для хранения партий
- маппинг между transport/domain/storage моделями

## Статус

Это не полноценно реализованный сервис, а скорее аккуратно оформленная архитектурная основа. Оставляю ее в портфолио как пример того, как я продумываю структуру backend-приложения до начала реализации.
